

------- DIR
1. dir(module_name): Возвращает список атрибутов, определенных в модуле module_name.
2. Без аргументов dir()возвращает список атрибутов из текущей области определений.

# main.py 
from mymodule import say_hello as greeting
print ( dir ())
 print (greeting( "World" ))

Вывод: [ '__annotations__' , '__builtins__' , '__cached__' , '__doc__' , '__file__' , '__loader__' , ' __name__ ' , ' __package__' , '__spec__' , 'greeting' ]
Hello, World!



-------   SYS

1. sys.argv список аргументов командной строки, переданных скрипту Python. Элемент argv[0]cявляется именем скрипта, а другие элементы – дополнительные аргументы командной строки.
2. sys.exit() – функция выхода из Python. Вы можете передать числовой довод, который станет статусом выхода программы. Принято, что аргумент 0означает успешное завершение, а ненулевые значения указывают на ошибку.
3. sys.path список строк, определяющий путь поиска интерпретатора для модулей. Вы можете изменить этот список для добавления собственных способов поиска модулей.
4. sys.version — строка, содержащая информацию о используемой версии Python.
5. sys.platform — строчка, указывающая на имя платформы, на которой выполняется Python (например, 'linux' для Linux, 'win32' для Windows).
6. sys.modules — словарь, содержащий загруженные модули. Ключи – это названия модулей, а значение – это объекты модулей.


Получить список модулей, встроенных в язык, можно с помощью переменной sys.builtin_module_names
import sys
 import os
print (sys.builtin_module_names)

При импорте модуля Python полагается на список путей для его поиска. Этот список хранится в переменной sys.path. Чтобы проверить, по каким путям интерпретатор будет искать модули, выведите sys.path.
