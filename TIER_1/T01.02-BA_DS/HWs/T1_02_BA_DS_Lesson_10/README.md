## Завдання 2. Обчислення визначеного інтеграла методом Монте-Карло

### Умова
Обчислити значення визначеного інтеграла методом Монте-Карло (площа під графіком) та перевірити результат, порівнявши його з `scipy.integrate.quad`

### Вхідні дані
- Функція: `f(x) = x^2`
- Межі інтегрування: `a = 0`, `b = 2`

### Візуалізація
Побудовано графік функції `f(x) = x^2` та заштриховано область під кривою на відрізку `[a, b]` (сіра зона)

### Метод Монте-Карло
1. Будується прямокутник: `x ∈ [a, b]`, `y ∈ [0, y_max]`, де `y_max = f(b)`
2. Генеруються випадкові точки всередині прямокутника
3. Рахується частка точок, що потрапили під криву: `M / N`
4. Площа під кривою оцінюється як:  
   `S ≈ (M / N) * (площа прямокутника)`
5. Експеримент повторюється кілька разів, результат усереднюється.

Для відтворюваності результату встановлено фіксований seed:
- `random.seed(42)`

Параметри симуляції:
- `num_points = 15000` (точок в одному експерименті)
- `num_experiments = 100` (кількість повторів)

### Перевірка через SciPy quad
Для перевірки використано функцію `quad`, яка обчислює інтеграл більш точним чисельним методом та повертає оцінку абсолютної похибки

### Результати (з фіксованим seed=42)
- Monte Carlo integral: `2.6606026666666662`
- quad integral: `2.666666666666667`
- quad abs error estimate: `2.960594732333751e-14`
- Absolute difference |MC - quad|: `0.0060640000000007355`

### Висновок
Метод Монте-Карло дав значення, яке відрізняється від результату `quad` на ≈ `6.064e-3` для вибраних параметрів та фіксованого `seed=42`
Оцінка похибки `quad` (≈ `2.96e-14`) стосується точності самого чисельного методу `quad`, тоді як відхилення Монте-Карло визначається випадковою природою методу та залежить від `num_points` і `num_experiments`
Щоб зменшити різницю між Монте-Карло та `quad`, потрібно збільшити `num_points` та/або `num_experiments`
